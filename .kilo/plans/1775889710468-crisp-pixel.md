# Plan: Multi-Agent Labor Division & Recruitment Efficiency

## Overview
Implement labor division mechanism for orders with 2+ agents using percentage-based allocation stored in the agents table. Display per-agent progress bars on order detail page.

## Requirements

### 1. Database Schema Changes

**agents table:**
- Add column `labor_percentage` (integer, nullable, default null)
  - Range: 0-100
  - Only used when order has 2+ agents
  - Manually edited by admin
  - Example: Agent A = 50%, Agent B = 50%

### 2. Type Definitions (lib/types.ts)

**Update Agent interface:**
```typescript
export interface Agent {
  id: string;
  supabase_uid: string | null;
  full_name: string | null;
  short_name: string | null;
  role: string | null;
  avatar_url: string | null;
  labor_percentage: number | null;  // NEW
}
```

### 3. Order Detail Page Updates (app/order/[id]/page.tsx)

**Data Fetching:**
- Fetch order data (already done)
- Fetch candidates for the order (already done)
- **NEW**: Fetch all agents assigned to this order via `agent_ids` array
- **NEW**: Fetch each agent's `labor_percentage` from agents table
- Calculate each agent's allocated labor: `allocated_labor = (labor_percentage / 100) * total_labor`
- Count passed candidates per agent: filter candidates by `agent_id` and `interview_status === 'Passed'`

**UI Changes:**
- Replace single "Recruitment Efficiency" section with per-agent progress bars
- For each agent assigned to the order:
  - Display agent avatar (if available) or initials
  - Display agent name (short_name or full_name)
  - Display allocated labor count
  - Display passed candidates count
  - Display progress bar: `passed / allocated_labor`
  - Display percentage completed

**Validation Display:**
- If any agent has null `labor_percentage` on multi-agent order:
  - Show error message: "Vui lòng nhập % lao động cho tất cả agent"
  - Hide progress bars until all percentages are set
- If percentages don't sum to 100%:
  - Show warning message (but still display progress bars)

**Logic:**
- If order has 1 agent: Show single progress bar (current behavior)
- If order has 2+ agents:
  - Fetch all agents from `agent_ids` array
  - For each agent: calculate allocated labor and passed count
  - Display per-agent progress bars
  - No overall progress bar

### 5. Edge Cases & Validation

**Validation:**
- Ensure `labor_percentage` is between 0-100
- For multi-agent orders, **block save** if percentages don't sum to 100%
- **Require manual input** for agents with null `labor_percentage` on multi-agent orders
- Show error message if validation fails

**Display Logic:**
- If `labor_percentage` is null for an agent on multi-agent order:
  - Show error: "Vui lòng nhập % lao động cho tất cả agent"
  - Block display of progress bars until all percentages are set
- Only show per-agent progress bars (no overall bar)

## Data Flow

**Order Detail Page:**
1. Fetch order data (includes agent_ids array)
2. Fetch all candidates for the order
3. For each agent in agent_ids:
   - Fetch agent data (including labor_percentage)
   - Calculate allocated_labor = (labor_percentage / 100) * total_labor
   - Count passed candidates = filter(candidates, agent_id == this.id && interview_status == 'Passed')
4. Render per-agent progress bars

**Admin Agent Page:**
1. Fetch agent data (including labor_percentage)
2. Display labor_percentage in form
3. On change, auto-save with 1.5s debounce
4. Validate 0-100 range before save

## Decisions Made

1. **Percentage Validation**: Block save if percentages don't sum to 100%
2. **Null Percentage Handling**: Require manual input, show error if null
3. **Overall Progress**: No, only show per-agent progress bars
