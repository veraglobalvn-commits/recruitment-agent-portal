### A. BẢO MẬT — 🔴 NGHIÊM TRỌNG

|---|---|
| `n8n/[Portal API] Get Dashboard Data.json:27` | Lark `app_id` + `app_secret` |

**Hậu quả:** Nếu repo public, ai cũng có thể lấy credentials để truy cập Lark Suite API, gọi Google Vision/Gemini tốn tiền. Ngay cả repo private, việc này vi phạm nguyên tắc security cơ bản.

**Khuyến nghị:**
- Move tất cả secrets vào `.env` hoặc n8n Credentials Store
- Thêm `n8n/*.json` vào `.gitignore` (hoặc dùng `.env` references trong workflow)
- Nếu đã push lên remote, **ROTATE tất cả keys ngay lập tức**

#### 2. Không có `.env.example`
Code dùng 5 env variables nhưng không có file hướng dẫn. Người khác clone repo sẽ không biết cần cấu hình gì.

#### 3. Không có `next.config.*`
Thiếu file config — không có headers bảo mật, không có image domains whitelist, không có redirect rules.

#### 4. Không có authentication middleware
Portal chỉ check login bằng state `isLoggedIn` (client-side). Không có middleware bảo vệ route `/order/[id]` — user chưa login vẫn có thể truy cập trực tiếp bằng URL nếu biết orderId.

### B. N8N WORKFLOWS — ⚠️ CẦN SỬA

#### Workflow ③: Upload Passport — 🔴 Lỗi kết nối song song

```
Webhook ──┬── Get Lark Token ────── Google Vision OCR
           └── Làm sạch Base64 ────/
```

Cả 2 nhánh đều nối vào `Google Vision OCR`. Node này nhận 2 items:
- Item 1 từ `Get Lark Token`: có `tenant_access_token`, **KHÔNG CÓ** `clean_base64`
- Item 2 từ `Làm sạch Base64`: có `clean_base64`, **KHÔNG CÓ** `tenant_access_token`

→ `{{ $json.clean_base64 }}` sẽ rỗng cho item 1 → **OCR fail**

**Sửa:** Đổi flow thành tuần tự: `Webhook` → `Get Lark Token` → `Làm sạch Base64` → `Google Vision OCR`

#### Không có Pagination (Workflow 1 & 2)

Lark Bitable API mặc định chỉ trả **100 records/page**. Khi data vượt 100 records:
- Dashboard sẽ thiếu orders/agents
- Candidate list sẽ thiếu ứng viên

**Sửa:** Thêm loop với `page_token` cho tất cả Lark API calls.

#### Không có Error Handling

Tất cả workflow đều không có error handler node. Nếu một node fail, webhook không trả response → frontend bị treo loading vô hạn.

**Sửa:** Thêm Error Trigger node hoặc try/catch trong Code nodes.

#### Upload Passport: Base64 qua JSON body

Gửi ảnh base64 qua JSON body có giới hạn ~10MB (do n8n/web server limits). Hộ chiếu scan thường 2-5MB, nén JPEG 0.8 thì OK nhưng không room cho margin.

### C. FRONTEND — ⚠️ CẦN CẢI TIẾN

#### 1. Kiến trúc "God Component" (file page.tsx)
`app/page.tsx` (154 dòng) chứa: Login form + Dashboard stats + Orders list + Pie chart — tất cả trong 1 component.

**Vấn đề:**
- Khó bảo trì, khó test
- Re-render toàn bộ khi 1 state thay đổi
- Logic nghiệp vụ lẫn lộn với UI

**Khuyến nghị:** Tách thành các components: `<LoginForm />`, `<DashboardStats />`, `<PaymentChart />`, `<OrdersList />`, `<OrderCard />`

#### 2. Không có TypeScript types
Tất cả data đều dùng `any[]`, `any`, `null`:
```ts
const [stats, setStats] = useState<any>(null);
const [orders, setOrders] = useState<any[]>([]);
```

**Khuyến nghị:** Định nghĩa interfaces `Agent`, `Order`, `Candidate`, `DashboardStats`

#### 3. Nút "Passed/Failed" trên trang Order Detail không hoạt động
`app/order/[id]/page.tsx:131-132` — Hai nút này chỉ là `<button>` không có `onClick` handler. Không update gì cả.

#### 4. Nút "View All Candidates", "Demand Letter", "List" không hoạt động
- `page.tsx:147`: "View All Candidates" không có `Link` hay handler
- `order/[id]/page.tsx:126-127`: "Demand Letter" và "List" chỉ là button rỗng

#### 5. Login state chỉ ở client-side
`isLoggedIn` là React state, mất khi refresh. Supabase có `getSession()` nhưng không được dùng. Cần check session khi app mount.

#### 6. Không có loading skeleton, error boundary
Chỉ có text "Loading data..." đơn giản. Không có Error Boundary để catch runtime errors.

#### 7. Không responsive tốt trên mobile
Layout dùng `md:grid-cols-2` nhưng nhiều phần chưa tối ưu cho màn hình nhỏ.
